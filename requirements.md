# フリークライマー向けアプリケーション MVP仕様書

## 1. MVP概要

### プロジェクト名
**ClimbTracker** - シンプルなクライミング記録アプリ

### MVP目的
認証機能なしで、個人のクライミング記録を手軽に管理できるWebアプリケーションを開発する。ローカルストレージを使用してブラウザ内でデータを保存し、基本的な記録・表示機能を提供する。

### MVPのスコープ
- ✅ 登攀記録の追加・表示・編集・削除
- ✅ 統計情報の表示
- ✅ レスポンシブデザイン
- ❌ ユーザー認証（後のフェーズで実装）
- ❌ ソーシャル機能（後のフェーズで実装）
- ❌ 外部API連携（後のフェーズで実装）

## 2. 技術仕様（MVP版）

### フロントエンド
- **フレームワーク**: Next.js 14 (App Router)
- **UI コンポーネント**: shadcn/ui
- **スタイリング**: Tailwind CSS
- **状態管理**: React Context + useReducer
- **データ保存**: localStorage
- **フォーム管理**: React Hook Form + Zod
- **日付処理**: date-fns
- **アイコン**: Lucide React

### データ保存
- **ローカルストレージ**: ブラウザのlocalStorageを使用
- **データ形式**: JSON形式で保存
- **データ永続化**: ブラウザ内での永続化（単一デバイス）

## 3. MVP機能要件

### 3.1 登攀記録管理（CRUD機能）

#### 3.1.1 記録追加機能
- **基本情報入力**:
  - ルート名（必須）
  - クライミングエリア/ジム名（必須）
  - グレード（5.8, V2, 6a等）（必須）
  - ルートタイプ（ボルダー/リード/トップロープ）（必須）
  - 登攀日（デフォルト：今日）（必須）
  - 完登ステータス（完登/失敗/練習）（必須）
- **詳細情報**:
  - メモ・感想（任意）
  - 体感難易度（★1-5）（任意）
  - かかった時間（任意）

#### 3.1.2 記録表示機能
- **一覧表示**:
  - 記録の時系列表示（新しい順）
  - カード形式で見やすく表示
  - ページネーションまたは無限スクロール
- **詳細表示**:
  - 個別記録の詳細表示
  - 編集・削除ボタンの設置

#### 3.1.3 記録編集・削除機能
- **編集機能**: 既存記録の修正
- **削除機能**: 記録の削除（確認ダイアログ付き）

### 3.2 検索・フィルタリング機能

#### 3.2.1 基本検索
- **テキスト検索**: ルート名・エリア名での検索
- **フィルタリング**:
  - グレード範囲
  - ルートタイプ
  - 完登ステータス
  - 期間指定（1週間、1ヶ月、3ヶ月、全期間）

### 3.3 統計・分析機能

#### 3.3.1 基本統計
- **サマリー情報**:
  - 総記録数
  - 完登数・完登率
  - 最高グレード
  - 最近の活動（今週・今月の記録数）
- **グレード別統計**:
  - グレード別の記録数
  - グレード別の完登率
- **ルートタイプ別統計**:
  - ボルダー/リード/トップロープ別の記録数
- **時系列統計**:
  - 月別の活動量グラフ
  - 進歩の可視化

## 4. データ構造（localStorage）

### 4.1 ClimbRecord型定義

```typescript
interface ClimbRecord {
  id: string; // UUID
  routeName: string;
  area: string;
  grade: string;
  routeType: 'boulder' | 'lead' | 'toprope';
  date: string; // ISO 8601 format
  status: 'completed' | 'failed' | 'practice';
  notes?: string;
  rating?: number; // 1-5 stars
  duration?: number; // minutes
  createdAt: string; // ISO 8601 format
  updatedAt: string; // ISO 8601 format
}
```

### 4.2 ストレージキー
- `climb-records`: ClimbRecord[]の配列
- `app-settings`: アプリケーション設定（テーマ、表示設定等）

## 5. UI/UX設計（MVP版）

### 5.1 画面構成

#### 5.1.1 ダッシュボード（ホーム画面）
- **ヘッダー**: アプリ名、ナビゲーションメニュー
- **統計サマリー**: 基本統計をカード形式で表示
- **最近の記録**: 直近5件の記録をカード表示
- **クイックアクション**: "新しい記録を追加"ボタン

#### 5.1.2 記録一覧画面
- **検索・フィルターバー**: テキスト検索、基本フィルター
- **記録カード**: グリッド形式での記録表示
- **ページネーション**: 大量データ対応

#### 5.1.3 記録追加・編集画面
- **入力フォーム**: 必須項目と任意項目を区別
- **バリデーション**: リアルタイムエラー表示
- **保存・キャンセルボタン**: 適切なフィードバック

#### 5.1.4 統計画面
- **チャート表示**: グレード分布、月別活動量
- **数値サマリー**: 各種統計値の表示
- **期間フィルター**: 表示期間の調整

### 5.2 レスポンシブ対応
- **モバイル（320px〜768px）**: 縦積みレイアウト、大きなタップエリア
- **タブレット（768px〜1024px）**: 2カラムレイアウト
- **デスクトップ（1024px〜）**: 3カラムレイアウト、サイドバーナビ

- ソーシャル機能追加時のデータ共有対応
